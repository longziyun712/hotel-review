# 功能规格：花园酒店评论浏览网页

**功能分支**: `001-hotel-review-browser`  
**创建时间**: 2026-04-13  
**状态**: 草稿  
**输入**: 用户描述："基于 public 文件夹下的 enriched_comments.csv 花园酒店住客评论数据集，搭建一个评论浏览网页，要能按条件筛选、关键词搜索、可以查看图片，并且网页要美观。"

## 用户场景与测试 *(必填)*

### 用户故事 1 - 评论浏览与分页 (优先级: P1)

用户打开网页后，可以浏览所有 2542 条住客评论，以分页卡片形式展示，每页 12 条，包含评分、日期、出行类型、房型、评论内容和图片缩略图。

**为何此优先级**: 核心展示能力，无此功能则其他功能无意义。

**独立测试**: 打开 index.html，无需任何操作即可看到评论卡片列表并翻页。

**验收场景**:

1. **Given** 页面加载完成，**When** 用户进入网页，**Then** 展示前 12 条评论卡片，显示分页控件
2. **Given** 已有评论卡片，**When** 用户点击"下一页"，**Then** 加载下一批评论，页面平滑滚动至顶部
3. **Given** 评论正文超过 200 字，**When** 卡片默认渲染，**Then** 折叠显示前 4 行，有"展开全文"按钮

---

### 用户故事 2 - 多维度筛选 (优先级: P1)

用户可通过侧边栏筛选面板，按评分（1–5 分）、出行类型（家庭亲子/商务出差等）、房间类型（大床房/双床房/套房/主题房）和评论分类标签进行多维度组合筛选。

**为何此优先级**: 2542 条评论中快速定位目标群体的核心需求。

**独立测试**: 选择"家庭亲子"+ "套房"后，仅显示符合这两个条件的评论。

**验收场景**:

1. **Given** 筛选面板已加载，**When** 用户选择出行类型"商务出差"，**Then** 仅显示 travel_type=商务出差 的评论
2. **Given** 已选多个筛选条件，**When** 用户点击"清除所有筛选"，**Then** 所有条件重置，显示全部评论
3. **Given** 组合筛选结果为空，**When** 没有评论匹配，**Then** 显示空状态提示，引导调整条件

---

### 用户故事 3 - 关键词搜索 (优先级: P1)

用户在顶部搜索框输入关键词，实时过滤匹配的评论，并在评论文本中高亮显示匹配词。

**为何此优先级**: 自由文本搜索是评论系统的基础能力。

**独立测试**: 搜索"早餐"，结果仅含"早餐"的评论，且文中"早餐"被高亮。

**验收场景**:

1. **Given** 用户输入关键词，**When** 250ms 防抖后，**Then** 自动过滤显示包含该词的评论
2. **Given** 搜索结果存在，**When** 渲染卡片，**Then** 评论文本中的匹配词以黄色 `<mark>` 高亮
3. **Given** 关键词无匹配，**When** 没有评论包含该词，**Then** 显示空状态提示

---

### 用户故事 4 - 图片查看 (优先级: P2)

用户点击评论卡片中的图片缩略图，进入全屏灯箱模式查看大图，支持前后翻页、键盘控制和点击背景关闭。

**为何此优先级**: 图片是评论的重要内容，但不是页面可用的前提条件。

**独立测试**: 点击含图评论的缩略图，进入灯箱，查看大图并用键盘左右翻页。

**验收场景**:

1. **Given** 评论含有图片，**When** 用户点击缩略图，**Then** 打开灯箱展示该图片，背景变暗
2. **Given** 灯箱已打开，**When** 用户按键盘左/右方向键，**Then** 切换到上/下一张图片
3. **Given** 灯箱已打开，**When** 用户按 ESC 或点击背景遮罩，**Then** 灯箱关闭，页面恢复正常滚动

---

### 边界情况

- 评论文本含特殊 HTML 字符（`<`, `>`, `&`），是否正确转义？
- 图片 URL 来自第三方 CDN（携程），加载失败时显示灰色占位符（图标 + "图片加载失败"文字），保持缩略图尺寸不变，避免布局塌陷
- CSV 数据中 images 字段为空数组 `[]` 时，不显示图片区域
- 评分为非整数（如 4.5）时，如何渲染星级？
- 搜索词含正则特殊字符（如 `.`、`*`）时，是否安全处理？

## 需求 *(必填)*

### 功能需求

- **FR-001**: 系统 MUST 在页面加载时通过 `fetch` 加载 `./enriched_comments.csv` 并用 PapaParse 解析
- **FR-002**: 系统 MUST 支持按评分（全部/5/4/3/2分以下）单选筛选
- **FR-003**: 系统 MUST 支持按出行类型、房型、评论分类标签多选筛选（各维度间 AND 关系）
- **FR-004**: 系统 MUST 提供关键词实时搜索，250ms 防抖，仅搜索 `comment` 字段，结果高亮匹配词；搜索结果计数显示"共找到 X 条评论"
- **FR-005**: 系统 MUST 支持排序（最新/最早/评分高低/质量分/有用数）
- **FR-006**: 系统 MUST 以分页方式展示结果，每页 12 条
- **FR-007**: 系统 MUST 为含图评论提供灯箱查看，支持同组图片切换；图片加载失败时 MUST 显示灰色占位符（保持缩略图尺寸，含错误图标与"图片加载失败"文字）
- **FR-008**: 系统 MUST 长评论（>200字）默认折叠，用户可手动展开
- **FR-009**: 系统 MUST 在顶部展示统计摘要（总评论数、平均评分、含图评论数）
- **FR-010**: 系统 MUST 使用纯静态实现（HTML + CSS + Vanilla JS），无需构建步骤
- **FR-011**: 系统 MUST 支持键盘操作主要功能（灯箱：ESC 关闭、方向键切图；分页：Tab 可聚焦翻页按钮）；无需实现完整 WCAG 2.1 AA
- **FR-012**: 筛选与排序状态不做 URL 持久化，页面刷新后恢复默认状态（最新发布、无筛选）

### 关键实体

- **评论 (Review)**: id、comment文本、images数组、score(0.5–5.0)、publish_date、fuzzy_room_type、travel_type、useful_count、quality_score(0–10)、categories数组
- **筛选状态 (FilterState)**: search关键词、score段、travel集合、room集合、cats集合、sort排序键
- **灯箱状态 (LightboxState)**: 当前图片组、当前索引

- Q: 携程 CDN 图片加载失败时如何处理？→ A: 显示灰色占位符（图标 + "图片加载失败"），保持缩略图尺寸，避免布局塌陷

## 假设

- 用户使用现代浏览器（支持 ES2020、CSS Grid、Fetch API）
- 数据来源为静态 CSV，不需要实时更新或后端 API
- 图片 URL 托管于携程 CDN（dimg04.c-ctrip.com），访问依赖网络连通性
- 移动端响应式为 P2 优化项，不阻塞桌面端交付
- 无需用户认证或数据写入功能
- categories 字段为 Python list 字面量格式，需用 `replace("'", '"')` 转换后 JSON 解析 [NEEDS CLARIFICATION: 是否所有行均遵循此格式？]
- 搜索范围仅限评论正文（`comment` 字段），不跨其他字段搜索；跨字段定位需通过侧边栏筛选面板实现
